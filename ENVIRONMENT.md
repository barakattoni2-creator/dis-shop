# Environment Variables

Every environment variable the application reads, grouped by subsystem. This list was generated by
cross-referencing every `process.env.X` reference in the source tree (`pages/`, `services/`, `lib/`,
`components/`, `hooks/`, `features/`) against `.env.example` â€” nothing here is guessed from memory.

Copy `.env.example` to `.env.local` for local development. On Vercel, set these under
**Project Settings â†’ Environment Variables** (see `DEPLOYMENT.md`).

Variables prefixed `NEXT_PUBLIC_` are inlined into the client bundle at build time and are visible to
anyone viewing the site's source â€” never put a secret behind that prefix. Every other variable here is
server-only and never sent to the browser.

## Required for the site to function at all

| Variable | Required | Notes |
|---|---|---|
| `DATABASE_URL` | Yes | Postgres connection string (Neon/Supabase). Must use a **pooled** connection endpoint (e.g. Neon's `-pooler` host) â€” serverless functions open a fresh connection per invocation and a direct (non-pooled) connection limit will be exhausted quickly. Needed at **build time** too: several pages use `getStaticProps` and query the database during `next build`. |
| `SESSION_SECRET` | Yes | Long random string used to sign admin session cookies (HMAC). Generate with `node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"`. Rotating this invalidates all existing admin sessions. |
| `ADMIN_EMAIL` | Yes | Break-glass Super Admin identity, used only if no matching row exists in `AdminUser` (first deploy, or DB unreachable). |
| `ADMIN_PASSWORD_HASH` | Yes | Bcrypt hash of the break-glass admin password. Generate with `npm run hash-password -- "your-chosen-password"` â€” never put the raw password in an env var. |

## Build-vs-runtime note for Vercel

Because several pages are statically generated (`getStaticProps`) and query the database at build time,
`DATABASE_URL` (and any other variable a build-time code path reads) must be set for **all** environments
you build in â€” Production, Preview, *and* Development â€” under Vercel's environment variable scope
checkboxes, not just Production. A variable only set for "Production" will cause Preview-branch builds to
fail or silently fall back to placeholder data.
